Техническое задание для проекта Factory
Описание проекта
Factory это CRM система для организации производства нанесений(принтов) на
текстильных изделиях. Система состоит из программной части (сайта, развернутого на
виртуальной машине) и терминалов. Терминалы бывают клиентские и служебные.
Клиентский терминал представляет собой сенсорный киоск (возможно оснащение
чековым принтером) или мобильное устройство с браузером, возможно брендирование.
Служебные терминалы - это мобильные устройства с браузером, возможно
подключение термопринтера для наклеек.
Далее под словом терминал понимается страница данной роли, редактирование –
добавление, удаление, изменение, изменение количества.

Программное обеспечение должно представлять собой самодостаточный образ (со
всем настроенным дополнительным ПО) виртуальной машины на linux (Ubuntu) для
развертки на VirtualBox с ярлыками на рабочем столе (подойдут и редактируемые
скрипты) для быстрого и легкого изменения адреса ресурса и проброса портов, так же
для удобства дублирования. Таких ресурсов может быть несколько на разных машинах.
Должны быть проброшены порт 80 и 22 для ресурса и удаленного доступа
соответственно. Так же должен создан репозитарий на гитхабе.
Решение будет развернуто на сервере на виртуальных машинах в нескольких
экземплярах. В качестве терминалов будут использованы киоски, компьютеры
(изготовление принтов) и мобильные устройства. На данный момент использовать

термопринтер можем только в ос Windows, и киоски с win соответственно. Браузер уже
настроен для киоска с печатью без превью. На термопринтере киоска печатаем только
текст с номером заказа и прочей информацией.
Список используемых понятий
Цвет – именованное значение цвета.
Модель – характеризуется цветом картинками спереди и сзади и уникальным названием.
Т.е. при выборе цвета можно получить список моделей данного цвета.
Размер – именованное буквенное уникальное значение (3XS – 6XL)
Существует связь между моделью и размером – 2 картинки с размерами. Т.е. при
выборе модели можно получить список всех доступных для этой модели размеров. А
при выборе модели и размера можно получить изображения изделия.
При наложении нанесения на модель должен будет соблюдаться масштаб. Из картинок
нанесений и модели с размером можно получить их размеры для правильного
масштабирования.
Описание страниц рабочих мест.
Оформление
При загрузке и после 10 минутной неактивности включается картинка заставки (по
умолчанию c содержанием “нажмите для работы”).
После прикосновения появляется интерфейс работы. Интерфейс должен быть
пошаговым со стрелками вперед/назад до завершения шага подтверждения, на первом
шаге стрелка назад не отображается, на шаге подтверждения вперед не отображается.
При выборе позиции на каждом шаге осуществляется переход на следующий.
Первый шаг – ВЫБЕРИТЕ ЦВЕТ. Если во всех доступных в наличии моделях цвет один,
то этот шаг автоматически пропускается и в качестве выбора будет зафиксирован
единственный цвет. Должна быть показана дискретная палитра цветов всех доступных в
наличии моделей. Цвета должны быть представлены большими фигурами с механизмом
свайпа для тачскринов.
Второй шаг – ВЫБЕРИТЕ МОДЕЛЬ. Если выбранного цвета доступная в наличии модель
одна, то этот шаг автоматически пропускается и в качестве выбора будет
зафиксирована единственная модель. Должен быть показан доступный в наличии
модельный ряд с картинками и названиями с механизмом свайпа для тачскринов.
Третий шаг – ВЫБЕРИТЕ РАЗМЕР. Если для выбранной модели в наличии только один
размер, то этот шаг автоматически пропускается и в качестве выбора будет
зафиксирован единственный в наличии размер. Размерный ряд должен отображаться
фигурами с надписью размеров, доступных в наличии с механизмом свайпа для
тачскринов.
Четвертый шаг – ВЫБЕРИТЕ ПРИНТ. На половине области экрана (примерно квадрат
для широкоформатных дисплеев) должна быть отображено изделие и область
нанесения (прямоугольник). На другой половине должна быть текстовая информация о
модели и размере, область превью всех доступных принтов для выбора с механизмом
свайпа для тачскринов. На первых местах всегда находятся особые принты «Свой текст»
(они отличаются цветом). При выборе принта он должен появиться в области нанесения
и за нее не выходить. Для обычных принтов пользователю доступны действия

переместить и повернуть, для текстовых – переместить, повернуть, выбрать шрифт
(один из доступных, один на весь текст), изменить текст.
Пятый шаг – ПОДТВЕРЖДЕНИЕ. На этом шаге пользователю доступны интерфейс для
ввода ФИО (как к вам обращаться), телефона. Так же пользователь выбирает один из
способов уведомления о готовности: смс, письмо на почту, WhatsApp, viber, telegram, не
уведомлять. При выборе способа показать соответствующий интерфейс ввода
контактной информации. После подтверждения формируется заказ, пользователю
передается номер заказа на экране для фото на телефон и при наличии принтера
печатается. После нажатия на экран терминал переходит на первый шаг и все
выбранные значения сбрасываются.
После оформления у заказа появляется статус ПОДТВЕРЖДЕНИЕ.
Подтверждение
Оператору доступна таблица со всеми заказами со статусом ПОДТВЕРЖДЕНИЕ, а
также строка для поиска Он может подтвердить заказ, произведя поиск по номеру/
имени/ телефону. Он выбирает найденную строку, где будет информация о номере
заказа, цвете, модели, размере, имени и телефоне посетителя. Оператор может
отменить заказ если посетитель пожелает (в случае, если передумает). При выполнении
действия подтвердить принтер должен распечатать лист с номером заказа, превью
стороны с принтом и информацию о цвете, размере, модели. После подтверждения
страница должна обновится.
После подтверждения у заказа появляется статус НАНЕСЕНИЕ или ПЕЧАТЬ, в
зависимости от наличия параметра изготовления нанесения у сессии.
Нанесение
Это звено задействуется, только тогда, когда АДМИНИСТРАТОР задал для сессии
производство нанесений, в противном случае оно не участвует (и отсутствие нанесений
«свой текст» в превью доступных принтов, т.к. они изготавливаются МАСТЕРОМ
ПРИНТОВ).
В интерфейсе должна быть большая кнопка ПОЛУЧИТЬ ЗАДАНИЕ. При её нажатии,
если есть заказы со статусом НАНЕСЕНИЕ, но не назначенным исполнителем,
открывается новая вкладка с заданием и назначается текущий исполнитель. В ней
оператору должен быть виден номер заказа, превью нанесения, размер нанесения
(ширина, высота), (номер нанесения если это не «свой текст»), и (если «свой текст» текст
нанесения, шрифт). По завершению изготовления принта оператор нажимает кнопку
ГОТОВО на его странице заказа, и она закрывается. После того, как заказ отправляется
на выполнение этому оператору, другим МАСТЕРАМ ПРИНТОВ будет недоступен. Затем
он получает следующий заказ на странице с кнопкой ПОЛУЧИТЬ ЗАДАНИЕ. У каждого
заказа должен быть зафиксирован один МАСТЕР ПРИНТА.
После нажатия кнопки ГОТОВО у заказа меняется статус на ПЕЧАТЬ.
Печать
В интерфейсе МАСТЕРА ПЕЧАТИ
В интерфейсе должна быть большая кнопка получить задание. При её нажатии, если
есть заказы на печать (со статусом ПЕЧАТЬ, без заданного исполнителя), интерфейс
печати, в котором должен быть номер заказа, сторона печати, общие виды готового
изделия со всех сторон с принтом и областью печати, сдвиг левого верхнего угла

нанесения относительно верхнего левого угла области печати и угол поворота против
часовой стрелки, и кнопка ГОТОВО.
По завершению печати оператор нажимает кнопку ГОТОВО на странице заказа, у заказа
меняется статус на ВЫДАЧА и страница открывает следующий заказ, либо большую
кнопку ПОЛУЧИТЬ ЗАДАНИЕ, если заказы для печати отсутствуют. После того, как
заказ отправляется на выполнение этому оператору, другим МАСТЕРАМ ПЕЧАТИ этот
заказ будет недоступен. У каждого заказа должен быть зафиксирован один МАСТЕР
ПЕЧАТИ.
После нажатия кнопки ГОТОВО у заказа меняется статус на ВЫДАЧА.
Выдача
Оператору доступна таблица с заказами со статусом ВЫДАЧА, а также строка для
поиска Он может ВЫДАТЬ заказ, произведя поиск по номеру/ имени/ телефону. Он
выбирает найденную строку, где будет информация о номере заказа, цвете, модели,
размере, имени и телефоне посетителя. После подтверждения страница должна
обновится. После подтверждения у заказа появляется статус ЗАВЕРШЕНО.
Табло
Табло, показывающее список номеров готовых (со статусом ВЫДАЧА) и в обработке (со
статусами ПОДТВЕРЖДЕНИЕ, НАНЕСЕНИЕ и ПЕЧАТЬ) заказов. Не имеет интерфейсов
ввода.
